# ВкладНавигатор — ИИ-ассистент потребителя финансовых услуг (подбор вклада)

Прототип ассистента, который помогает клиенту банка **принять информированное решение
о вкладе**: по заданным параметрам он строит сравнительную таблицу вариантов, считает
реальный доход с учётом капитализации, выделяет скрытые условия и риски и **корректно
отказывается**, когда запрос невыполним или вне его компетенции.

Проект сделан как **MVP полного жизненного цикла модели (ЖЦМ)** в продуктовой среде:
данные → ядро решения → объяснение LLM → валидация → развёртывание → мониторинг →
противодействие деградации → критерии вывода из эксплуатации.

```
        ┌──────────────────────────────────────────────────────────────────┐
        │  Веб-форма (Streamlit): сумма, срок, валюта, цель, пополнение/    │
        │  снятие, ожидаемая ставка, свободный вопрос                       │
        └───────────────────────────────┬──────────────────────────────────┘
                                         ▼
   ┌─────────────────────────────────────────────────────────────────────────┐
   │  ДЕТЕРМИНИРОВАННОЕ ЯДРО  (src/core) — источник истины по всем цифрам      │
   │   1. guardrails / отказ   (guards.py)   ← корректный отказ               │
   │   2. жёсткая фильтрация    (filtering.py)                                │
   │   3. эффективная ставка    (effective_rate.py)  ← капитализация, APY     │
   │   4. ранжирование под цель (ranking.py)                                  │
   │   → сравнительная таблица top-N + риск-флаги (АСВ, досрочное закрытие…)  │
   └───────────────────────────────┬─────────────────────────────────────────┘
                                    │ структурированный результат (числа)
                                    ▼
   ┌─────────────────────────────────────────────────────────────────────────┐
   │  LLM-СЛОЙ  (src/llm, GigaChat-2-Max) — объяснение на естественном языке  │
   │   • пересказывает результат ядра, НЕ вводит новых чисел                  │
   │   • groundedness-проверка: при галлюцинации → откат к шаблону            │
   │   • офлайн-режим без ключа: детерминированные объяснения                 │
   └───────────────────────────────┬─────────────────────────────────────────┘
                                    │ каждый ответ → журнал (JSONL)
                                    ▼
   ┌─────────────────────────────────────────────────────────────────────────┐
   │  МОНИТОРИНГ И УПРАВЛЕНИЕ МОДЕЛЬЮ  (src/monitoring, src/validation)        │
   │   дрейф (PSI) · свежесть каталога (SLA) · доля галлюцинаций ·            │
   │   валидация на golden-наборе · критерии вывода модели из эксплуатации    │
   └─────────────────────────────────────────────────────────────────────────┘
```

## С чего начать (для ревью и второго участника)

Порядок чтения, чтобы за 15–20 минут понять проект целиком:

1. **Этот README** — что за проект, архитектура, соответствие критериям (вы здесь).
2. [`docs/repo_guide.md`](docs/repo_guide.md) — **навигация по коду**: что где лежит,
   поток данных, как запускать. Главная карта репозитория.
3. Запустить локально (см. «Установка» и «Запуск» ниже): `make install && make data`,
   потом `make app` — открыть http://localhost:8501 и потыкать вкладку «🎬 Демо».
4. Прогнать проверки: `make test` (42 теста), `make eval` (валидация → вердикт OK),
   `make drift` (демонстрация деградации → REFRESH → восстановление).
5. [`presentation/roles.md`](presentation/roles.md) — **кто что делал** и кто что
   рассказывает на защите; [`presentation/script.md`](presentation/script.md) — текст
   доклада по слайдам + вопросы комиссии; [`presentation/demo_runbook.md`](presentation/demo_runbook.md)
   — **пошаговый запуск, токен GigaChat и тайминг демо**;
   [`presentation/deck.html`](presentation/deck.html) — **готовые слайды** (открыть в
   браузере, печать в PDF через Cmd/Ctrl+P, альбомная).
6. Глубже по каждому критерию — `docs/00_overview.md` … `docs/06_decommission.md`.

Где «сердце» проекта в коде: `src/core/` (детерминированное ядро — все цифры),
`src/llm/` (объяснение + защита от галлюцинаций), `src/monitoring/` (эксплуатация).

## Что умеет ассистент

- **Сравнительная таблица** вкладов под параметры клиента с расчётом эффективной ставки.
  Пример скрытого условия: вклад с номинальной ставкой **17,9 %** и ежемесячной
  капитализацией даёт эффективную **19,44 %** — ядро считает это детерминированно.
- **Риск-флаги**: превышение лимита страхования АСВ (1,4 млн ₽), сгорание процентов при
  досрочном закрытии, отсутствие капитализации, акционные условия, только-онлайн.
- **Корректные отказы** (8 кодов): некорректный ввод, вне компетенции, завышенные
  ожидания (анти-мисселинг), сумма/срок/валюта вне каталога, prompt injection.

### Подключение GigaChat (LLM-слой)

LLM **не обязателен**: без ключа ассистент выдаёт детерминированные объяснения, и всё
демо (включая отказы, валидацию, мониторинг) работает полностью. Чтобы включить
объяснения от GigaChat-2-Max:

```bash
cp .env.example .env
# в .env указать ключ авторизации из личного кабинета developers.sber.ru:
#   GIGACHAT_CREDENTIALS=<Base64 client_id:client_secret>
#   GIGACHAT_MODEL=GigaChat-2-Max
```

Даже при включённом LLM показанный клиенту текст **всегда обоснован числами ядра**:
если модель «придумывает» цифру, ответ отбраковывается (groundedness-проверка) и
заменяется детерминированным шаблоном, а инцидент фиксируется в журнале.

## Результаты валидации (воспроизводимо: `make eval`, `make drift`, `make test`)

- **Golden-набор (17 кейсов):** пройдено 17/17; точность ранжирования top-1 = **1.0**
  (MRR = 1.0); отказы precision/recall/F1 = **1.0** (confusion: tp 8 / fp 0 / fn 0 / tn 9),
  точность кода отказа = 1.0. Вердикт эксплуатации: **OK**.
- **Симуляция дрейфа:** при снижении ключевой ставки и устаревшем каталоге дрейф ставок
  значимый (PSI ≫ 0,25; MAE по ставке ≈ 5,4 п.п.), точность top-1 относительно реального
  рынка падает 1.0 → **0.8**, свежесть 107 дней (critical) → вердикт **REFRESH**; после
  обновления каталога метрики восстанавливаются (top-1 = 1.0, вердикт OK).
- **Тесты:** 42 passed (формулы APY, guardrails/отказы, фильтрация, ранжирование,
  groundedness, метрики, дрейф, критерии вывода модели).

## Учебные данные

Каталог вкладов сгенерирован детерминированно: ставка каждого продукта привязана к
**ключевой ставке ЦБ** (`ставка = base(ключевая) + надбавка за срок + надбавка архетипа`).
Это и реалистично, и удобно для демонстрации дрейфа: сдвиг ключевой ставки порождает
согласованный сдвиг всего каталога. Названия банков вымышленные. Данные — учебные и не
являются офертой.
